# 📖 Introdução

---

# 🛠️ Descrição do Projeto

A implementação foi dividida em 3 etapas principais, cada uma utilizando um serviço AWS específico para atender necessidades distintas da empresa.

---

## ✅ Etapa 1 — Amazon EC2

### 🎯 Foco da Ferramenta
Redução de custos com infraestrutura física e aumento da escalabilidade dos servidores.

### 📌 Caso de Uso
A empresa possuía servidores locais com altos custos de manutenção e baixa flexibilidade.  
Com a migração para o **Amazon EC2**, foi possível:

- Provisionar servidores sob demanda;
- Reduzir gastos com hardware físico;
- Melhorar a disponibilidade das aplicações;
- Escalar recursos conforme necessidade do negócio.

### 🔧 Benefícios Obtidos
- Redução de custos operacionais;
- Maior disponibilidade dos sistemas;
- Escalabilidade sob demanda.

---

## ✅ Etapa 2 — Amazon S3

### 🎯 Foco da Ferramenta
Armazenamento seguro e econômico de arquivos corporativos.

### 📌 Caso de Uso
Os arquivos da empresa eram armazenados em servidores locais, gerando altos custos de backup e manutenção.

Com a implementação do **Amazon S3**, foi possível:

- Centralizar o armazenamento de documentos;
- Melhorar a durabilidade e segurança dos dados;
- Automatizar backups;
- Reduzir custos com infraestrutura de armazenamento.

### 🔧 Benefícios Obtidos
- Alta durabilidade dos dados;
- Backup automatizado;
- Redução de custos com armazenamento físico.

---

## ✅ Etapa 3 — AWS Lambda

### 🎯 Foco da Ferramenta
Automação de processos e redução de custos com execução de serviços.

### 📌 Caso de Uso
Processos internos executados continuamente consumiam recursos computacionais desnecessários.

Com o uso do **AWS Lambda**, foi possível:

- Executar funções sob demanda;
- Eliminar custos com servidores ociosos;
- Automatizar tarefas administrativas;
- Melhorar a eficiência operacional.

### 🔧 Benefícios Obtidos
- Modelo de cobrança por execução;
- Menor consumo de recursos;
- Automação de processos internos.
